# The Knowledge Game

A mobile feed you can point at a community of typed, graded claims: scroll it, question it, contribute to it, or found a community of your own. The first app for the [EpiStack protocol](https://github.com/A-Viable-Fork/epistack).

The kernel runs on your device. When this app shows you a claim's standing, nothing decided it for you: your phone fetched the community's snapshot, verified its hash, and recomputed every grade from the claim's own support structure. There is no server that could lie to you, because there is no server.

**Try it:** https://a-viable-fork.github.io/Knowledge-Game/ works in any modern browser and installs as an app from the browser menu. Browsing requires no account, ever.

## What is different here

**The algorithm answers to you.** The feed's ranking objective is a visible vector you set: learn efficiently, strongest opposition, unresolved questions, recent changes, or your own weighting. Every card answers "why am I seeing this." Turn ranking off and the feed drops to a declared deterministic order. Behavioral observation is opt-in and off by default; off means not collected. Ranking provably cannot touch a grade: a mechanical check fuzzes arbitrary objectives and asserts every grade and certificate byte-identical.

**Standing is computed, never voted.** There are no likes and no popularity-derived grades. A claim's grade recomputes from what supports it, on your device, and comments are a separate thing entirely: discussion threads that carry no grade and can never be cited as support, so talking never masquerades as evidence.

**Three states, never conflated.** Gate-passed means a contribution's structure holds. Admitted means a community took it into its store. Semantically accepted means people, attributably, judged it sound. No screen in this app renders one as another, and a mechanical check enforces the wording.

**This client is unprivileged, provably.** It holds no capability any client with the snapshot lacks. A minimal independent client, built in CI from the vendored public api alone, reproduces every capability of this app against the same published artifacts, reading and writing. Communities founded here contain no reference to this app; a check greps for it. If you can build a better client, nothing stops you, including us.

**Extensions load through a theorem.** Third-party rankers and renderers install by content hash and run sandboxed; install-time conformance runs the same fuzz that binds our own ranker, so an extension that could move a grade does not load. Our default ranker ships as the first extension, through the same public path.

## The repository accounts for itself

This repo carries a governance kernel generated by the protocol's own tooling. Its claims describe this app's behavior: ranking cannot modify standing, profile data cannot enter a public patch, no undeclared network egress, the periphery imports no kernel module, and the rest. Every claim was entered bare at its honest floor and earned its grade when a named check landed; the gate computed every standing shown, none was assigned by hand. The commit history walks the protocol's six-stage workflow and `trellis/stage-manifest.json` is the machine-checked record. What is not yet verified is characterized in `trellis/sorry-ledger.md` with closing conditions, and `trellis/status-ledger.md` draws the built/specified line per item.

## Architecture in brief

One trust boundary, one direction: `periphery/` -> `api/` -> vendored kernel, statically enforced. The upstream substrate is pinned by commit and hash-locked (`upstream/lock.json`, verified on every build). Public kernel data and private personal data are structurally separate: everything durable about you lives in `vault/`, local, exportable, deletable, and a fuzz check proves vault contents cannot serialize into a public contribution. Four seams are built as explicit interfaces with honest null implementations awaiting upstream: credentials and identity presentation, durable patch history, live remote providers, and the standing economy.

## Run, verify, fork

Local: clone with `--recurse-submodules`, serve the repo root with any static file server, open `app/`. No build step exists; what you read is what ships.

Verify: `node build/check-substrate.mjs` proves the vendored substrate matches the lock; the full `build/check-*.mjs` suite is what CI runs; `manifests/` carries build provenance and the artifact hash. A hash proves identity, not good behavior; the governance claims and their checks are the behavior story.

## The first community

The EpiStack Competition Community lives at `communities/epistack-competition/`: claims about the protocol itself, mechanical ones grounded by upstream's own checks, evaluative ones sitting at their honest floor awaiting the only thing that can lift them, which is people deciding they hold. Its card, snapshot, and contribution path are in the directory; contributions arrive as pull requests whose gate re-runs in Actions.

This repository standardizes what must be shared for its artifacts to compose. It does not standardize what should remain local: personal objectives, semantic judgment, participant governance, or community purpose. Founding a community means setting those, and the founding flow is that sentence made into a screen.
